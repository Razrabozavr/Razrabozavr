# Политика конфиденциальности Трекер неровностей

**Последнее обновление:** 3 июля 2026 г.

## 1. Общие положения

Настоящая Политика конфиденциальности («Политика») описывает, какие данные собирает, обрабатывает и хранит мобильное приложение **Трекер неровностей** («Приложение»), а также как пользователь может управлять своими данными.

### 1.1. Цель настоящей Политики

Защита Вашей конфиденциальности чрезвычайно важна для Разработчика. Настоящая Политика призвана прозрачно проинформировать Вас о следующих моментах:

- какие персональные данные собираются и обрабатываются при использовании Приложения;
- как и зачем Разработчик обрабатывает Ваши персональные данные;
- какие меры защиты применяются для обеспечения конфиденциальности;
- какие права Вы имеете в отношении своих персональных данных;
- как Вы можете управлять своими данными и отзывать согласие.

### 1.2. Принятие Политики

Используя Приложение, Вы подтверждаете, что:

- ознакомились с настоящей Политикой и согласны с условиями обработки персональных данных;
- предоставление персональных данных осуществляется исключительно на Вашей свободной воле;
- Вы не обязаны предоставлять какие-либо персональные данные, однако без предоставления необходимых данных (геолокация, акселерометр) использование Приложения невозможно;
- Вы осознаёте, что отказ в предоставлении данных делает использование Приложения невозможным.

Если Вы не согласны с какими-либо положениями, пожалуйста, не используйте Приложение.

## 2. Какие данные собираются

### 2.1. Данные о местоположении

Приложение запрашивает разрешения для работы с геолокацией:

- точное местоположение (GPS);
- приблизительное местоположение (сеть).

**Цель сбора:** запись GPS-треков, отображение текущей позиции на карте, привязка данных акселерометра к координатам.

Для непрерывной записи треков при свёрнутом приложении используется фоновый режим работы. В зависимости от версии операционной системы может потребоваться дополнительное разрешение, которое запрашивается отдельно при первом запуске.

Пользователь может в любой момент отозвать разрешение на доступ к местоположению в настройках операционной системы, однако это приведёт к остановке записи треков.

### 2.2. Данные с датчиков устройства

Приложение использует встроенный **акселерометр** для:

- определения вибраций и неровностей дорожного покрытия;
- вычисления индекса неровности.

**Сырые данные акселерометра не покидают устройство** и не передаются третьим лицам. На их основе вычисляется индекс неровности, который сохраняется в локальной базе данных вместе с GPS-координатами (см. раздел 2.3).

### 2.3. Данные, создаваемые пользователем

В процессе использования Приложение создаёт:

- GPS-треки (координаты, временные метки, индекс неровности);
- настройки пользователя (тема, интервал GPS, пороги чувствительности).

Пользователь также может импортировать треки из внешних файлов в локальную базу данных Приложения (см. раздел 4.5).

### 2.4. Технические данные

Приложение использует интернет **только** для загрузки картографических тайлов и проверки состояния сети.

**Приложение НЕ отправляет на серверы Разработчика:**

- данные о местоположении;
- данные акселерометра;
- созданные треки;
- настройки пользователя;
- данные об использовании (аналитику);
- персональные данные (имя, email, телефон);
- рекламные идентификаторы.

**Вся информация, создаваемая Приложением, остаётся на устройстве пользователя.**

При загрузке картографических тайлов передаются стандартные технические данные, необходимые для работы интернет-сервисов (подробнее в разделе 9.1).

### 2.5. Что Разработчик НЕ делает

Разработчик **НЕ**:

- проверяет достоверность предоставленных Вами данных;
- собирает чувствительную персональную информацию (расовое происхождение, политические взгляды, информация о здоровье, биометрические данные);
- использует Ваши данные для целей, не указанных в настоящей Политике;
- продаёт или сдаёт в аренду Ваши данные третьим лицам.

### 2.6. Случаи передачи данных третьим лицам

Приложение передаёт минимально необходимые данные следующим третьим сторонам исключительно для выполнения своих функций:

**Картографические сервисы** (раздел 9.1):

- координаты видимой области карты — для загрузки картографических тайлов

**Сервисы геолокации** (раздел 9.2):

- технические данные о местоположении — для определения координат устройства

## 3. Как используются данные

### 3.1. Цели обработки

Все собранные данные используются **исключительно** для:

- отображения треков на карте в реальном времени;
- сохранения истории поездок в локальной базе данных;
- анализа неровности дорожного покрытия;
- предоставления пользователю возможности экспорта и импорта данных.

Данные **НЕ используются** для:

- отслеживания местоположения пользователя в коммерческих целях;
- передачи третьим лицам в коммерческих целях;
- таргетированной рекламы;
- профилирования поведения.

### 3.2. Добровольность предоставления данных

Предоставление персональных данных осуществляется исключительно на Вашей свободной воле и в Вашем интересе для использования функционала Приложения. Предоставление данных не является обязанностью по закону, однако необходимо для работы Приложения.

Вы вправе в любой момент отозвать согласие на обработку персональных данных (раздел 7.3), однако это приведёт к невозможности дальнейшего использования Приложения.

## 4. Хранение и защита данных

### 4.1. Локальное хранение

В локальную базу данных Приложения сохраняются только следующие данные:

- геопозиция (координаты маршрута);
- индекс неровности;
- временные метки.

Настройки приложения хранятся отдельно от базы данных треков.

### 4.2. Срок хранения данных

Срок хранения данных: до момента удаления пользователем или удаления Приложения. Пользователь может удалить данные в любой момент через встроенный механизм (раздел 4.6). Разработчик не осуществляет автоматической очистки устаревших треков.

### 4.3. Резервное копирование

База данных Приложения может включаться в стандартный механизм резервного копирования Android (Google Drive). Это означает, что копия данных может автоматически сохраняться в облачное хранилище в соответствии с настройками Вашего устройства.

**Важно:** резервное копирование — это функция операционной системы Android, а не Приложения. Приложение лишь разрешает системе создавать резервные копии своих данных. Управление резервным копированием (включение/отключение, очистка бэкапов) осуществляется пользователем через настройки устройства:

**Настройки → Система → Резервное копирование**

Разработчик не имеет доступа к резервным копиям, не управляет ими и не несёт ответственности за данные, восстановленные из резервных копий.

### 4.4. Экспорт данных

Пользователь может экспортировать свои треки через встроенную функцию «Экспорт». Экспортированные файлы содержат координаты, временные метки и индекс неровности. Настройки приложения в экспорт не включаются.

**В момент экспорта данные покидают Приложение.** Дальнейшие действия с экспортированными файлами (копирование, перемещение, передача третьим лицам, загрузка в облачные сервисы) полностью контролируются пользователем.

Разработчик **не несёт ответственности** за сохранность экспортированных файлов, их использование или передачу третьим лицам, а также за последствия, связанные с размещением экспортированных данных в сторонних сервисах.

### 4.5. Импорт данных

Приложение предоставляет функции импорта треков из внешних файлов в локальную базу данных.

**Доступны два режима импорта:**

**Режим 1: Импорт (с заменой)**  
Удаляет все существующие треки из локальной базы данных и импортирует новые записи из выбранного файла.

**Режим 2: Добавление (без замены)**  
Добавляет новые треки из выбранного файла в локальную базу данных без удаления существующих записей.

**Важно:**

- Импорт данных осуществляется **исключительно в локальную базу данных** Приложения на устройстве пользователя.
- Импортированные данные **не передаются** на серверы Разработчика или третьих лиц.
- Разработчик **не имеет доступа** к импортированным файлам и не контролирует их содержание.
- Пользователь несёт полную ответственность за источник и содержание импортируемых файлов.
- Разработчик **не несёт ответственности** за достоверность, точность или законность импортированных данных.

### 4.6. Удаление данных

**Приложение предоставляет встроенные механизмы удаления данных из своей локальной базы.**

Пользователь может самостоятельно удалить данные следующими способами:

**Способ 1: удаление конкретных треков**

Настройки → Редактирование треков.  
Данная функция позволяет выбрать и удалить отдельные треки из базы данных.

**Способ 2: полная очистка базы данных**

Настройки → Очистить базу данных.  
Данная функция удаляет все треки в базе данных (включая импортированные).

**Способ 3:** удалить Приложение через настройки операционной системы.  
При стандартном удалении приложения локальная база данных уничтожается вместе с ним.

**Важно:** Приложение удаляет только данные, находящиеся в его локальной базе данных. Приложение **не может удалить** экспортированные файлы и резервные копии, созданные системой Android. Удаление этих данных осуществляется пользователем самостоятельно.

## 5. Меры защиты персональных данных

### 5.1. Технические меры

- **Локальное хранение:** Приложение хранит все данные на Вашем устройстве в защищённой базе данных, доступной только Приложению.
- **Шифрование:** База данных использует встроенные механизмы шифрования Android.
- **Изоляция данных:** Приложение не передаёт данные на внешние серверы Разработчика.
- **Минимизация данных:** Приложение собирает только те данные, которые необходимы для выполнения основных функций.

### 5.2. Организационные меры

- **Ограничение доступа:** Разработчик не имеет физического или технического доступа к данным пользователей.
- **Принцип минимизации:** Приложение не запрашивает избыточных разрешений или данных.

### 5.3. Ответственность пользователя

Поскольку данные хранятся на Вашем устройстве, Вы несёте ответственность за:

- защиту устройства от несанкционированного доступа (пароль, биометрия);
- регулярное резервное копирование данных (через функцию экспорта);
- удаление экспортированных файлов, если они больше не нужны;
- содержание и источник импортируемых файлов.

## 6. Ваши права

### 6.1. Перечень прав

Вы имеете следующие права в отношении своих персональных данных:

**Право на доступ:**

- получить информацию о том, какие Ваши персональные данные обрабатываются;
- получить копию Ваших данных в структурированном формате (через функцию экспорта).

**Право на удаление:**

- требовать удаления Ваших персональных данных (раздел 7);
- отозвать согласие на обработку персональных данных (раздел 7.3).

**Право на ограничение обработки:**

- ограничить обработку Ваших данных путём отключения доступа к местоположению в настройках операционной системы.

**Дополнительные права:**

В случаях, предусмотренных применимым законодательством, Вы можете также обладать другими правами в отношении своих персональных данных, не указанными в настоящей Политике.

### 6.2. Как реализовать свои права

**Самостоятельно через интерфейс Приложения:**

- Удаление конкретных треков: Настройки → Редактирование треков
- Полная очистка базы данных: Настройки → Очистить базу данных
- Экспорт данных: Настройки → Экспорт
- Импорт данных: Настройки → Импорт
- Удаление приложения: Настройки устройства → Приложения → Трекер неровностей → Удалить

**Путём направления запроса Разработчику:**

- Контактные данные указаны в разделе 11 настоящей Политики
- Разработчик рассматривает запросы в течение 30 календарных дней
- Разработчик подтверждает выполнение запроса или предоставляет инструкцию по самостоятельному выполнению

## 7. Запросы пользователей

### 7.1. Порядок подачи запроса

Для подачи запроса на удаление данных или отзыв согласия на обработку персональных данных необходимо направить обращение по контактным данным, указанным в разделе 11.

В запросе достаточно указать:

- email, с которого направлен запрос;
- примерную дату последнего использования Приложения.

Разработчик рассматривает email-адрес отправителя как достаточную идентификацию субъекта персональных данных.

### 7.2. Обработка запросов

В течение 30 календарных дней Разработчик направляет пользователю ответ, содержащий:

- подтверждение факта обращения;
- ссылку на инструкцию по самостоятельному удалению данных (раздел 4.6);
- подтверждение того, что на стороне Разработчика обработка персональных данных не осуществлялась.

Направление инструкции является надлежащим исполнением обязательств Разработчика.

### 7.3. Прекращение обработки

Для полного прекращения обработки персональных данных пользователь должен удалить Приложение со своего устройства. После этого обработка данных на устройстве прекращается.

**Примечание:** Резервные копии, созданные системой Android (см. раздел 4.3), пользователь удаляет самостоятельно через настройки облачного хранилища.

**Обработка данных лиц младше 18 лет допускается только при наличии явного согласия родителей или законных представителей.**

Разработчик сознательно не собирает, не обрабатывает и не хранит персональные данные несовершеннолетних.

В случае обнаружения факта использования Приложения несовершеннолетним **без согласия родителей или законных представителей**, мы рекомендуем родителям или законным представителям:

- самостоятельно удалить все треки через встроенный механизм Приложения (Настройки → Очистить базу данных);
- удалить Приложение через настройки операционной системы.

Если родители или законные представители дают согласие на использование Приложения несовершеннолетним, они несут полную ответственность за контроль использования Приложения и защиту данных на устройстве.

## 9. Сторонние сервисы

### 9.1. Картографические сервисы

Приложение использует бесплатные картографические сервисы для отображения карт.

При каждом запросе к картографическому серверу передаются следующие данные:

**Передаётся:**

- **IP-адрес устройства** — технически неизбежен для любого HTTP-запроса;
- **User-Agent** — идентификатор приложения/браузера, может содержать информацию об устройстве и версии операционной системы;
- **Заголовки запроса** — язык устройства, кодировка и другие служебные параметры;
- **Координаты видимой области карты** — для загрузки картографических тайлов (не являются точными координатами местоположения устройства).

**Не передаётся:**

- идентификаторы устройств (IMEI, Android ID, Advertising ID);
- точные координаты местоположения устройства.

Политика конфиденциальности картографического сервиса:  
https://wiki.osmfoundation.org/wiki/Privacy_Policy

### 9.2. Сервисы геолокации

Для получения геолокации используются системные сервисы операционной системы. Эти сервисы могут использовать агрегированные анонимные данные для улучшения точности геопозиционирования. Разработчик не получает и не хранит эти промежуточные данные.

### 9.3. Трансграничная передача данных

При загрузке картографических данных может происходить передача данных на серверы, расположенные за пределами Российской Федерации. Эта передача осуществляется исключительно для получения картографических данных.

Подробнее о перечне передаваемых данных см. раздел 9.1.
