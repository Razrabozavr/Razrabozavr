# Политика конфиденциальности "Ежедневник. Мои задачи"

**Последнее обновление:** 3 июля 2026 г.  
**Версия:** 1.0

---

## 1. Общие положения

Настоящая Политика конфиденциальности («Политика») описывает, какие данные собирает, обрабатывает и хранит мобильное приложение "Ежедневник. Мои задачи" («Приложение»), а также как пользователь может управлять своими данными.

### 1.1. Цель настоящей Политики
Защита Вашей конфиденциальности чрезвычайно важна для Разработчика. Настоящая Политика призвана прозрачно проинформировать Вас о следующих моментах:
* какие данные собираются и обрабатываются при использовании Приложения;
* как и зачем Разработчик обрабатывает Ваши данные;
* какие меры защиты применяются для обеспечения конфиденциальности;
* какие права Вы имеете в отношении своих данных;
* как Вы можете управлять своими данными и отзывать согласие.

### 1.2. Принятие Политики
Используя Приложение, Вы подтверждаете, что:
* ознакомились с настоящей Политикой и согласны с условиями обработки данных;
* предоставление данных осуществляется исключительно на Вашей свободной воле;
* Вы не обязаны предоставлять какие-либо данные, однако без сохранения необходимых данных (текста задач, настроек) использование Приложения невозможно;
* Вы осознаёте, что отказ в сохранении данных делает использование Приложения невозможным.

Если Вы не согласны с какими-либо положениями, пожалуйста, не используйте Приложение.

## 2. Какие данные собираются

### 2.1. Данные, создаваемые пользователем
В процессе использования Приложение создаёт и хранит:
* текст задач и заметок;
* статусы задач и временные метки (дата создания, срок выполнения, дата завершения).

### 2.2. Настройки приложения
Приложение сохраняет Ваши предпочтения для обеспечения удобства работы:
* выбранная тема оформления (светлая, тёмная, системная);
* параметры масштабирования интерфейса.

### 2.3. Технические данные и сетевые запросы
Приложение использует интернет исключительно для проверки наличия обновлений. 
Приложение **не отправляет** на серверы Разработчика:
* созданные задачи и заметки;
* настройки пользователя;
* данные об использовании (аналитику);
* именные данные (ФИО, email, телефон);
* рекламные идентификаторы.

Технический IP-адрес устройства передаётся на сервер исключительно для проверки обновлений (подробнее в разделе 9.1). Вся остальная информация, создаваемая Приложением, остаётся на устройстве пользователя.

### 2.4. Что Разработчик НЕ делает
Разработчик **НЕ**:
* проверяет достоверность предоставленных Вами данных;
* собирает чувствительную информацию (расовое происхождение, политические взгляды, информация о здоровье, биометрические данные);
* использует Ваши данные для целей, не указанных в настоящей Политике;
* продаёт или сдаёт в аренду Ваши данные третьим лицам.

### 2.5. Случаи передачи данных третьим лицам
Приложение передаёт минимально необходимые данные следующим третьим сторонам исключительно для выполнения своих функций:
* **Сервисы проверки обновлений (раздел 9.1):**
  * технический IP-адрес и версия приложения — для информирования о наличии новых версий.

## 3. Как используются данные

### 3.1. Цели обработки
Все собранные данные используются исключительно для:
* отображения задач и управления ими в интерфейсе Приложения;
* сохранения пользовательских настроек между сеансами работы;
* предоставления актуальной информации о доступных обновлениях.

Данные **НЕ используются** для:
* отслеживания поведения пользователя в коммерческих целях;
* передачи третьим лицам в коммерческих целях;
* таргетированной рекламы;
* профилирования поведения.

### 3.2. Добровольность предоставления данных
Предоставление данных осуществляется исключительно на Вашей свободной воле и в Вашем интересе для использования функционала Приложения. 
Вы вправе в любой момент отозвать согласие на обработку данных (раздел 7.3), однако это приведёт к невозможности дальнейшего использования Приложения.

## 4. Хранение и защита данных

### 4.1. Локальное хранение
В локальную базу данных Приложения сохраняются только следующие данные:
* текст задач, статусы и временные метки;
* настройки оформления и интерфейса.

### 4.2. Срок хранения данных
Срок хранения данных: до момента удаления пользователем или удаления Приложения. Пользователь может удалить данные в любой момент через встроенный механизм (раздел 4.5). Разработчик не осуществляет автоматической очистки устаревших задач.

### 4.3. Резервное копирование
Приложение имеет встроенный механизм автоматического локального резервного копирования базы данных. Копия данных сохраняется в скрытую папку внутри памяти устройства. Это позволяет восстановить данные в случае их повреждения или сбоя.
Разработчик не имеет доступа к локальным резервным копиям, не управляет ими и не несёт ответственности за данные, хранящиеся в них.

### 4.4. Экспорт и импорт данных
Пользователь может экспортировать свои задачи через встроенную функцию «Экспорт». Экспортированные файлы содержат все данные из локальной базы.
В момент экспорта данные покидают Приложение. Дальнейшие действия с экспортированными файлами (копирование, перемещение, передача третьим лицам, загрузка в облачные сервисы) полностью контролируются пользователем.
Разработчик не несёт ответственности за сохранность экспортированных файлов, их использование или передачу третьим лицам.

Пользователь может импортировать ранее экспортированные файлы через встроенную функцию «Импорт». При импорте данные загружаются в локальную базу данных Приложения и остаются на устройстве. Разработчик не имеет доступа к импортируемым данным и не осуществляет их передачу на внешние серверы.

### 4.5. Удаление данных
Приложение предоставляет встроенные механизмы удаления данных из своей локальной базы.
Пользователь может самостоятельно удалить данные следующими способами:
* **Способ 1: удаление конкретных задач**
  Удаление задач по одной или группами через интерфейс Приложения.
* **Способ 2: удалить Приложение через настройки операционной системы.**
  При стандартном удалении приложения локальная база данных и скрытые папки резервного копирования уничтожаются вместе с ним.

*Важно: Приложение удаляет только данные, находящиеся в его локальной базе данных. Приложение не может удалить экспортированные файлы, сохранённые пользователем вручную. Удаление этих данных осуществляется пользователем самостоятельно.*

## 5. Меры защиты данных

### 5.1. Технические меры
* **Локальное хранение:** Приложение хранит все данные на Вашем устройстве в защищённой базе данных, доступной только Приложению.
* **Изоляция данных:** Приложение не передаёт задачи и настройки на внешние серверы Разработчика.
* **Минимизация данных:** Приложение собирает только те данные, которые необходимы для выполнения основных функций.

### 5.2. Организационные меры
* **Ограничение доступа:** Разработчик не имеет физического или технического доступа к задачам и настройкам пользователей.
* **Принцип минимизации:** Приложение не запрашивает избыточных разрешений или данных.

### 5.3. Ответственность пользователя
Поскольку данные хранятся на Вашем устройстве, Вы несёте ответственность за:
* защиту устройства от несанкционированного доступа (пароль, биометрия);
* регулярное создание резервных копий данных (через функцию экспорта);
* удаление экспортированных файлов, если они больше не нужны.

## 6. Ваши права

### 6.1. Перечень прав
Вы имеете следующие права в отношении своих данных:
* **Право на доступ:**
  * получить информацию о том, какие Ваши данные обрабатываются;
  * получить копию Ваших данных в структурированном формате (через функцию экспорта).
* **Право на удаление:**
  * требовать удаления Ваших данных (раздел 7);
  * отозвать согласие на обработку данных (раздел 7.3).
* **Право на ограничение обработки:**
  * ограничить обработку Ваших данных путём отключения доступа к интернету в настройках операционной системы (это отключит проверку обновлений, но сохранит основной функционал).

### 6.2. Как реализовать свои права
**Самостоятельно через интерфейс Приложения:**
* Удаление конкретных задач: через главный экран Приложения.
* Экспорт данных: через меню настроек.
* Удаление приложения: Настройки устройства → Приложения → "Ежедневник. Мои задачи" → Удалить.

**Путём направления запроса Разработчику:**
* Контактные данные указаны в разделе 11 настоящей Политики.
* Разработчик рассматривает запросы в течение 30 календарных дней.
* Разработчик подтверждает выполнение запроса или предоставляет инструкцию по самостоятельному выполнению.

## 7. Запросы пользователей

### 7.1. Порядок подачи запроса
Для подачи запроса на удаление данных или отзыв согласия на обработку данных необходимо направить обращение по контактным данным, указанным в разделе 11.
В запросе достаточно указать:
* email, с которого направлен запрос;
* примерную дату последнего использования Приложения.

Для обработки запроса мы используем email-адрес, с которого поступило обращение, в качестве контактных данных для направления ответа. В случае сомнений в принадлежности запроса, мы можем запросить дополнительную информацию для подтверждения личности.

### 7.2. Обработка запросов
В течение 30 календарных дней Разработчик направляет пользователю ответ, содержащий:
* подтверждение факта обращения;
* ссылку на инструкцию по самостоятельному удалению данных (раздел 4.5);
* подтверждение того, что на стороне Разработчика обработка данных не осуществлялась.

Направление инструкции является надлежащим исполнением обязательств Разработчика.

### 7.3. Прекращение обработки
Для полного прекращения обработки данных пользователь должен удалить Приложение со своего устройства. После этого обработка данных на устройстве прекращается.

В случае обнаружения факта использования Приложения несовершеннолетним без согласия родителей или законных представителей, мы рекомендуем родителям или законным представителям:
* самостоятельно удалить все задачи через интерфейс Приложения;

## 9. Сторонние сервисы

### 9.1. Сервисы проверки обновлений
Приложение использует сетевое соединение исключительно для проверки наличия новых версий. Вся передача данных осуществляется по защищённому протоколу **HTTPS** с использованием шифрования TLS.

При каждом запросе к серверу обновлений передаются следующие данные:
**Передаётся:**
* IP-адрес устройства — технически неизбежен для любого сетевого запроса (передаётся на сетевом уровне для маршрутизации пакетов);
* User-Agent — идентификатор приложения, может содержать информацию об устройстве и версии операционной системы;
* Текущая версия приложения — для определения необходимости обновления.

**Не передаётся:**
* идентификаторы устройств (IMEI, Android ID, Advertising ID);
* содержимое задач и заметок;
* настройки приложения;
* именные данные (ФИО, email, телефон).

### 9.2. Трансграничная передача данных
При проверке обновлений может происходить передача технических данных на серверы, расположенные за пределами Российской Федерации. Эта передача осуществляется исключительно для получения информации о версии приложения и защищена протоколом HTTPS.
Подробнее о перечне передаваемых данных см. раздел 9.1.

## 10. Обновление настоящей Политики

Разработчик может периодически обновлять настоящую Политику. При этом Разработчик обязуется:
* не вносить существенных изменений, налагающих дополнительные обременения или ограничения Ваших прав, без предварительного уведомления;
* уведомлять о существенных изменениях через:
  * обновление приложения с изменением версии Политики;
  * изменение даты «Последнее обновление» в начале документа.

Продолжение использования Приложения после внесения изменений означает Ваше согласие с новой редакцией Политики.
Если Вы не согласны с изменениями, Вы вправе:
* отозвать согласие на обработку данных (раздел 7.3);
* удалить Приложение со своего устройства.
