# Краткое содержание Политики конфиденциальности
## Ваши данные — только ваши 🔒

### • Разработчик не собирает ваши данные  
Приложение не запрашивает ФИО, email, телефон или адрес. Разработчик не знает, кто вы и что пишете в приложении.

### • Все задачи хранятся только у вас  
База данных находится на вашем телефоне. Разработчик не имеет к ней доступа ни технически, ни физически.

### • Приложение работает без интернета  
Основной функционал доступен офлайн. Интернет нужен только для проверки обновлений через RuStore.

### • Никакой аналитики и рекламы  
Разработчик не отслеживает, как вы используете приложение, не собирает статистику и не показывает таргетированную рекламу.

### • Вы полностью контролируете свои данные  
- Можете создавать, редактировать и удалять задачи.  
- Можете экспортировать все задачи в файл и сохранить где угодно.  
- Можете импортировать задачи из резервной копии.  
- Можете удалить приложение вместе со всеми данными через настройки телефона.

### • Автоматическое резервное копирование  
Приложение создаёт скрытую копию базы данных на вашем устройстве. Если данные повредятся, они восстановятся из бэкапа.

### • Если вы напишете разработчику  
Ваш email используется только для ответа на ваш запрос. После завершения переписки адрес удаляется, никаких рассылок и рекламы.

### • Приложение не для детей  
Это персональный инструмент для планирования, не рассчитанный на использование детьми. Родители сами контролируют использование приложения несовершеннолетними.

### • Обновления через RuStore  
При проверке обновлений передаются только технические данные (IP-адрес, версия приложения). Разработчик не управляет сбором данных магазином RuStore.

### • Политика может меняться  
Если условия обновятся, вы узнаете об этом при следующем запуске приложения. Продолжение использования означает согласие с новой редакцией.

## 1. Общие положения

### 1.1. Статус Разработчика
Разработчик приложения "Ежедневник. Мои задачи" **не является оператором персональных данных** в смысле Федерального закона № 152-ФЗ "О персональных данных".

Обоснование:
* Приложение не запрашивает и не собирает персональные данные пользователей (ФИО, email, телефон, адрес и иные подобные данные);
* Приложение работает преимущественно в автономном (оффлайн) режиме и не требует постоянного подключения к сети Интернет;
* Разработчик не имеет физического или технического доступа к установленному на устройстве пользователя Приложению, его локальной базе данных или файлам;
* Разработчик не осуществляет сбор, хранение, обработку или передачу персональных данных пользователей на свои серверы;
* Сетевое соединение используется исключительно для проверки наличия обновлений через инфраструктуру магазина приложений RuStore.

### 1.2. Цель настоящей Политики
Настоящая Политика информирует пользователей о том:
* какие данные создаются и хранятся локально на устройстве пользователя в процессе использования Приложения;
* как обеспечивается конфиденциальность данных пользователя;
* какие права имеет пользователь в отношении своих данных;
* как пользователь может управлять своими данными.

### 1.3. Принятие Политики
Используя Приложение, Вы подтверждаете, что:
* ознакомились с настоящей Политикой;
* понимаете, что все данные создаются и хранятся исключительно на Вашем устройстве;
* осознаёте, что Разработчик не имеет доступа к Вашим данным и не осуществляет их обработку;
* принимаете ответственность за сохранность данных на Вашем устройстве.

Если Вы не согласны с какими-либо положениями, пожалуйста, не используйте Приложение.

## 2. Какие данные создаются в Приложении

### 2.1. Данные, создаваемые пользователем
В процессе использования Приложение **создаёт и хранит локально на Вашем устройстве**:
* текст задач и заметок;
* статусы задач и временные метки (дата создания, срок выполнения, дата завершения).

**Важно:** Эти данные создаются исключительно Вами, хранятся только на Вашем устройстве и не передаются Разработчику или третьим лицам.

### 2.2. Настройки приложения
Приложение сохраняет Ваши предпочтения для обеспечения удобства работы:
* выбранная тема оформления (светлая, тёмная, системная);
* параметры масштабирования интерфейса.

Настройки хранятся локально на Вашем устройстве и не передаются третьим лицам.

### 2.3. Сетевые запросы и технические данные
Приложение использует подключение к сети Интернет **исключительно** для проверки наличия обновлений через магазин приложений RuStore.

**Приложение не отправляет на серверы Разработчика:**
* созданные задачи и заметки;
* настройки пользователя;
* данные об использовании (аналитику);
* именные данные (ФИО, email, телефон);
* рекламные идентификаторы;
* любые иные персональные данные,

— **за исключением адреса электронной почты, добровольно предоставляемого пользователем при направлении запроса Разработчику для получения ответа** (подробнее в разделе 7.2).

Для проверки обновлений используется **SDK (Software Development Kit) магазина приложений RuStore**. При этом передаются только технические данные, необходимые сервису RuStore для предоставления информации о наличии новых версий (подробнее в разделе 9.1).

Вся остальная информация, создаваемая Приложением, остаётся на устройстве пользователя.

### 2.4. Что Разработчик НЕ делает
Разработчик **НЕ**:
* запрашивает или собирает персональные данные пользователей;
* проверяет достоверность данных, создаваемых пользователем;
* имеет доступ к локальной базе данных Приложения на устройстве пользователя;
* собирает чувствительную информацию (расовое происхождение, политические взгляды, информация о здоровье, биометрические данные);
* использует данные пользователя для каких-либо целей;
* продаёт или передаёт данные пользователей третьим лицам.

### 2.5. Передача данных третьим лицам
Разработчик не передаёт данные пользователей третьим лицам.

Единственным случаем сетевого взаимодействия является проверка обновлений через магазин приложений RuStore, которая осуществляется в соответствии с политикой конфиденциальности RuStore (подробнее в разделе 9.1).

## 4. Хранение и защита данных

### 4.1. Локальное хранение
Все данные хранятся **исключительно в локальной памяти Вашего устройства**:
* текст задач, статусы и временные метки — в локальной базе данных;
* настройки оформления и интерфейса — в локальном хранилище настроек.

Разработчик не имеет серверов для хранения данных пользователей и не осуществляет облачную синхронизацию.

### 4.2. Срок хранения данных
Срок хранения данных: до момента удаления пользователем или удаления Приложения. Пользователь может удалить данные в любой момент через встроенный механизм (раздел 4.5).

### 4.3. Резервное копирование
Приложение имеет встроенный механизм автоматического локального резервного копирования базы данных. Копия данных сохраняется в скрытую папку внутри памяти устройства. Это позволяет восстановить данные в случае их повреждения или сбоя.

Разработчик не имеет доступа к локальным резервным копиям, не управляет ими и не несёт ответственности за данные, хранящиеся в них.

### 4.4. Экспорт и импорт данных
**Экспорт:**  
Пользователь может экспортировать базу данных с задачами через встроенную функцию «Экспорт». Экспортированные файлы содержат задачи, статусы и временные метки. **Настройки приложения в экспорт не включаются.**

В момент экспорта данные покидают Приложение. Дальнейшие действия с экспортированными файлами (копирование, перемещение, передача третьим лицам, загрузка в облачные сервисы) полностью контролируются пользователем.

Разработчик не несёт ответственности за сохранность экспортированных файлов, их использование или передачу третьим лицам.

**Импорт:**  
Пользователь может импортировать ранее экспортированные файлы через встроенную функцию «Импорт». При импорте данные загружаются в локальную базу данных Приложения и остаются на устройстве. **Настройки приложения при импорте не восстанавливаются и сохраняют текущие значения.**

Разработчик не имеет доступа к импортируемым данным и не осуществляет их передачу на внешние серверы.

### 4.5. Удаление данных
Приложение предоставляет встроенные механизмы удаления данных из своей локальной базы.

Пользователь может самостоятельно удалить данные следующими способами:

**Способ 1: удаление конкретных задач**  
Удаление задач по одной через интерфейс Приложения.

**Способ 2: удалить Приложение через настройки операционной системы.**  
При стандартном удалении приложения локальная база данных и скрытые папки резервного копирования уничтожаются вместе с ним.

*Важно: Приложение удаляет только данные, находящиеся в его локальной базе данных. Приложение не может удалить экспортированные файлы, сохранённые пользователем вручную. Удаление этих данных осуществляется пользователем самостоятельно.*

## 5. Меры защиты данных

### 5.1. Технические меры
* **Локальное хранение:** Приложение хранит все данные на Вашем устройстве в защищённой базе данных, доступной только Приложению.
* **Изоляция данных:** Разработчик не имеет серверов для хранения задач и настроек пользователей.
* **Минимизация данных:** Приложение не запрашивает избыточных разрешений или данных.

### 5.2. Организационные меры
* **Отсутствие доступа:** Разработчик не имеет физического или технического доступа к задачам и настройкам пользователей.
* **Принцип минимизации:** Приложение не запрашивает персональные данные и не осуществляет их сбор.

### 5.3. Ответственность пользователя
Поскольку данные хранятся на Вашем устройстве, Вы несёте ответственность за:
* защиту устройства от несанкционированного доступа (пароль, биометрия);
* регулярное создание резервных копий данных (через функцию экспорта);
* удаление экспортированных файлов, если они больше не нужны.

## 9. Сторонние сервисы

### 9.1. Магазин приложений RuStore и SDK обновлений
Приложение распространяется через магазин приложений RuStore. Для проверки наличия обновлений Приложение использует **SDK (Software Development Kit) магазина приложений RuStore**.

**Что передаётся при проверке обновлений:**  
При каждом запросе к инфраструктуре RuStore передаются только технические данные, необходимые для определения наличия новой версии Приложения:
* IP-адрес устройства — технически неизбежен для любого сетевого запроса (передаётся на сетевом уровне для маршрутизации пакетов);
* текущая версия приложения — для определения необходимости обновления;
* технические идентификаторы, необходимые для работы SDK RuStore.

**Что НЕ передаётся Разработчику:**
* содержимое задач и заметок;
* настройки приложения;
* именные данные (ФИО, email, телефон);
* персональные данные пользователя;
* данные об использовании Приложения.

Все сетевые взаимодействия с магазином приложений осуществляются по защищённому протоколу **HTTPS** с использованием шифрования TLS.

**Важно:** Разработчик не управляет сбором данных магазином приложений RuStore и не несёт ответственности за обработку данных, осуществляемую RuStore в соответствии с его собственной политикой конфиденциальности.

### 9.2. Трансграничная передача данных
Инфраструктура магазина приложений RuStore расположена на территории Российской Федерации. Трансграничная передача данных, если она осуществляется магазином приложений, регулируется политикой конфиденциальности RuStore.

Разработчик не управляет трансграничной передачей данных, осуществляемой магазином приложений, и не несёт за неё ответственности.
